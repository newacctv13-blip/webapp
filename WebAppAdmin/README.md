# WebAppAdmin — Панель управления заказами Telegram

## 1. Создание или получение токена бота

1. Откройте Telegram и найдите **BotFather** — `@BotFather`
2. Если **нужно создать нового бота**:
   - Напишите команду `/newbot`
   - Введите имя бота (например, `My Orders Bot`)
   - Введите username бота, заканчивающийся на `bot` (например, `MyOrdersBot`)
   - BotFather выдаст токен вида: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`
3. Если **бот уже есть**, получите токен командой `/token` → выберите бота → скопируйте токен
4. Откройте файл `config.py` и замените `TELEGRAM_BOT_TOKEN` на ваш токен:

```python
TELEGRAM_BOT_TOKEN = "ВАШ_ТОКЕН_СЮДА"
```

---

## 2. Создание группы и добавление бота

1. В Telegram нажмите **три полоски** (меню) → **Создать группу**
2. Назовите группу (например, `Заказы`)
3. На этапе добавления участников **напишите username вашего бота** (например, `@MyOrdersBot`)
4. Нажмите **Создать**

---

## 3. Назначение бота администратором группы

> **Важно:** бот может читать сообщения в группе, только если он назначен администратором.

1. Откройте созданную группу
2. Нажмите на название группы вверху → **Редактировать**
3. Выберите **Администраторы**
4. Нажмите **Добавить администратора**
5. Выберите вашего бота из списка
6. **Оставьте все права выключенными**, кроме одного — **Отправка сообщений** (Send Messages) — оно нужно для обратной связи
7. **Право «Читать сообщения» (Read Messages) даётся автоматически** всем администраторам, дополнительно включать ничего не нужно
8. Нажмите **Сохранить**

---

## 4. Проверка

1. Напишите в группу любое сообщение (бот его увидит)
2. Если бот должен отвечать — в `app.py` есть эндпоинт `/proxy/sendMessage`, через который можно отправлять ответы от бота

---

## 5. Формат сообщений с заказами

Сообщения должны содержать поля в формате `Название: значение`. Пример:

```
Имя: Иван Петров
Телефон: +7-999-123-45-67
Город: Москва
Состав заказа: Пицца 2 шт, Кола 1 л
Сумма: 1500 руб
Доставка: 300 руб
К оплате: 1800 руб
Время: 2024-06-19 15:30
```

Распознаются русские и английские названия полей:

| Русский | Английский |
|---|---|
| Имя / Имя: | Name |
| Телефон / Тел: | Phone / Tel |
| Город / Гор: | City |
| Состав заказа / Состав: | Items / Order |
| Сумма / Сум: | Total |
| Доставка / Дост: | Delivery |
| К оплате / Оплата / Итого: | To pay |
| Время / Дата: | Time / Date / Datetime |

---

## 6. Как это работает (интеграция)

Админка получает заказы **двумя способами одновременно**:

1. **Webhook (реальное время)** — локальный обработчик заказов (`worker/telegram_handler.py`) при получении нового заказа сразу пересылает его в админку через `/ingest`. Админка сохраняет заказ в БД и отправляет его в Telegram.

2. **Polling (резервный)** — каждые 10 секунд админка проверяет новые сообщения в Telegram через Bot API и парсит их. Если заказ уже сохранён через webhook — пропускает дубликат.

Все заказы отображаются в веб-интерфейсе с возможностью:
- Фильтрации по статусу
- Поиска по имени, телефону, городу
- Изменения статуса (Принят → В обработке → Доставлен)
- Отмены заказа

## 7. Запуск

```bash
pip install -r requirements.txt
python app.py
```

Откройте браузер: **http://127.0.0.1:5000**

При запуске через `start_local.cmd` или `start_online.py` админка запускается автоматически.
